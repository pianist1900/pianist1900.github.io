技术工具｜构建自己的网站

Posted by Will Gong on Thursday, September 11, 2025

一切从分享欲开始
做技术工作已然有些时间了，大多数的学习都是来源于网络，有感网友的强大，也愈发觉得开源的美妙。从当初学python参考“廖雪峰的网站”，到做数据分析图参考“炸鸡人的博客”，在学习的过程中，发现他们都是有一个自己的网站。伴随着知识增加，开始逐渐想分享自己解决的一些问题，一些领域知识，在考察了知乎、小红书、CNDS之后，发现自己还是适合最存粹的网站，自己建个简单明了、没有限制的网站，于是开始了自己的探索。本篇记录利用github page+Hugo构建数据自己的网站(Mac为例)。

Github pages+Hugo建站流程

  1. 前置条件

    • 一个 GitHub 账号，如没有需要自行注册github账号；
    • 本地安装 Git,由于我使用电脑是Mac，在terminal中输入git，如果没有安装会弹框提示，点击确认就行，相对容易；安装完成在terminal中输入git --version，输出类似信息:“git version 2.39.5 (Apple Git-154)“则说明安装成功，没有则需要检查安装；

  2. Hugo安装
    Github pages支持三个快捷的网站框架，jekyll、hexo、hugo：

    • jekyll使用ruby编写，有github支持，可以将markdown文件直接放到git仓库，github会自动生成网页文件，同时Github一直是一个亲ruby的社区，找了一下jekyll主题感觉便丑，可能是我没找对；
    • Hexo使用nodejs，Hexo提供了方便的部署命令，可以做到一条命令部署到github上，没找到太多的文档，放弃；
    • Hugo使用go语言开发，Hugo的官方文档写的还不错，方便学习，同时部署简洁。

    最后选择了Hugo。

    本地安装Hugo（brew是mac里安装包的一个管理软件）非常简单，我使用的是macOS，官方推荐用 homebrew方式:

    brew install hugo

    如果提示没有brew,需要自行安装homebrew，官网安装说明比较容易，有命令和pkg两种方式，这里我采用的是github下载pkg的方式，gui界面点点点就行。
    安装hugo完成后，在terminal中输入hugo version查看安装是否成功，输出类似的版本信息:“hugo v0.149.1+extended+withdeploy darwin/arm64，则说明安装成功

  3. 创建本地Hugo站点
    1.创建本地站点文件夹 在terminal中输入hugo new site test,test更换成想要的文件名：

    hugo new site test

    执行完成后出现如下图所示内容，注意这里的test文件是在你当前文件目录下一级生成一个test的文件夹，比如这个当前目录为/Desktop，那么执行完成后test文件夹是在/Desktop/test
    文件夹目录如下图所示，其中

    • config.toml为hugo配置文件，是主要的网站程序；
    • content是用于发布的网页文件目录md文件和html等
    • static是用于网页发布中的静态资源，如图片等；
    • themes是选用的主题，主题一般会有案例信息可以参考；
    • public/文件夹为最终部署的网站内容。 2.选择主题 Hugo没有默认主题，所以需要自己安装主题，参考hugo指南中看到，官网主题库见Hugo Themes,以官方案例 Ananke 主题为例,在termianl中依次输入：
    git init
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
echo "theme = 'ananke'" >> hugo.toml

    执行完成后，在terminal中输入hugo server，在http://localhost:1313/将会看到由主题生成的空白网站，如下图所示。 3.添加页面 以上完成后，开始写入自己的第一个页面，页面内容一般使用markdown书写，写完后以markdown形式保存在/test/content目录下，具体书写规则见markdown基本语法，以官网为例：

    +++
title = 'My First Post'
date = 2024-01-14T07:07:07+01:00
draft = true
+++
## Introduction

This is **bold** text, and this is *emphasized* text.

Visit the [Hugo](https://gohugo.io) website!

    这里需要注意的是，前面的+++必须得有，也可以是---内容 内容 内容---，+或者-并不会显示，写完保存。然后输入以下的任意一个命令，将内容发布到本地网站:

    hugo server --buildDrafts
hugo server -D

    打开网站，发现内容发布成功。

  4. 部署到 GitHub Pages
    在 GitHub 上新建一个发布仓库
    登陆Github，根据仓库构建指南,构建一个用于发布的username.github.io，这里只能是usename，其它名称没法在公开网站上显示；
    建设完成后，在新的网页界面输入username.github.io会显示空白网页，如果提示404，请参考指南进行修正。
    建立本地推送sh文件
    基于bash，写一个本地推送的脚本保存在test目录下:

    #!/bin/bash
echo -e "\033[0;32mDeploying updates to GitHub Pages...\033[0m"
# 1. 生成静态文件（清理旧文件，使用 config 里的主题配置即可）
hugo --cleanDestinationDir
# 2. 进入 public 文件夹
cd public || exit
# 3. 初始化 git 仓库（只对 public 生效）
if [ ! -d .git ]; then
git init
git remote add origin git@github.com:pianist1900/pianist1900.github.io.git
fi
# 4. 切换到 main 分支
git checkout -B main
# 5. 添加改动
git add .
# 6. 提交
msg="rebuilding site $(date)"
if [ $# -eq 1 ]
then
msg="$1"
fi
git commit -m "$msg"
# 7. 强制推送到远程 main 分支
git push -f origin main
# 8. 回到项目根目录
cd ..

    本地推送
    在terminal中输入chmod +x deploy.sh && ./deploy.sh推送本地网站内容到GitHub Pages上， Github查看 完成后打开Github Pages确认是否内容更新，如有更新，几分钟内即可通过https://username.github.io/ 访问网站。

    值得注意的是，我这里的Github推送采用的是Github ssh进行推送，具体ssh构建与配置流程见github ssh流程。

常见问题

- 网页不可见：检查是否推送到正确分支，通常为main，并确认 Pages 设置；
- 文章不显示：确保文章在正确的目录下，部分主题目录不对，则不显示；
- 主题无效：确认 config.toml 中 theme 名称与主题目录一致，config.toml一定要加主题信息；
- 推送不成功： ssh配置一定得完成才行
